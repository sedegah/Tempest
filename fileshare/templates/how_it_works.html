The Workflow

Upload → Share → Vanish.

A secure, automated pipeline from encrypted file upload to permanent, verifiable destruction.

01
Upload & Encrypt
User submits a file via form or API. Optionally encrypted with AES-256. Django saves to Cloudflare R2 and stores metadata in the database.
  • Optional AES-256 encryption
  • Stored in Cloudflare R2
  • Metadata saved to DB
02
Generate Unique Link
Django generates a UUID-based link with a configurable expiry time. The link is returned to the uploader to share with the recipient.
  • SHA-256 obfuscated token
  • Expiry time configured
  • Optional password set
03
Validate & Serve
Django validates the token — checking expiry, download count, and password — then serves the decrypted file to the recipient.
  • Token + expiry + password checked
  • File decrypted on-the-fly
  • Rate limiting enforced
04
Destruct & Log
Link is instantly invalidated. Celery auto-deletes expired files from storage and the database. Every access is logged for auditing.
  • Link instantly invalidated
  • Celery removes from storage + DB
  • Access logged to database
