Frequently Asked Questions

Everything you need to know about Tempest.

When uploading, you can set a download limit (from 1 to 100). Tempest tracks the download_count, and once it reaches your specified limit, the link is permanently invalidated (410 Gone).
Encryption is optional and uses the Python cryptography library with AES-256 (Fernet). A unique key is generated per file, stored in the DB, and used to decrypt on-the-fly when the file is served.
You can upload files up to 100MB. We also restrict certain executable file types (.exe, .bat, .sh, .bin, .cmd, .msi) for security reasons.
You can choose an expiry period ranging from 1 hour to 1 month. After this time, the file and all associated metadata are permanently deleted from our storage and database.
You have 3 attempts to enter the correct password. If all 3 attempts fail, the file is instantly destroyed to prevent brute-force attacks.
Yes. When a file expires or is deleted (manually or via failed password attempts), it is purged from our Cloudflare R2 storage and the database record is removed. There is no way to recover it.
